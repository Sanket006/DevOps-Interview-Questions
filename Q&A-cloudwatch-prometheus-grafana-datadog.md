## ðŸ”¹ AWS CloudWatch (1â€“10)

### 1. What is Amazon CloudWatch?

CloudWatch is AWSâ€™s native monitoring and observability service. It collects metrics, logs, and events from AWS resources and applications, allowing you to monitor performance, detect anomalies, and trigger automated actions using alarms.

### 2. What types of data does CloudWatch collect?

CloudWatch collects metrics (CPU, memory, disk), logs (application and system logs), events (resource state changes), and traces (via AWS X-Ray integration).

### 3. Difference between CloudWatch Metrics and Logs?

Metrics are numerical time-series data (e.g., CPU utilization), while logs are unstructured or semi-structured text data (e.g., application logs). Metrics are used for monitoring trends; logs are used for troubleshooting.

### 4. What are CloudWatch Alarms?

CloudWatch Alarms monitor metrics and trigger actions when thresholds are breached, such as sending SNS notifications, auto-scaling EC2 instances, or stopping/terminating resources.

### 5. What is CloudWatch Logs Insights?

Logs Insights is a query tool that allows you to analyze logs using a SQL-like syntax. It helps in debugging, root cause analysis, and performance monitoring.

### 6. What are Custom Metrics in CloudWatch?

Custom metrics allow you to publish application-specific or system-level metrics not provided by default, such as memory usage or business KPIs.

### 7. How does CloudWatch integrate with Auto Scaling?

CloudWatch alarms trigger scaling policies based on metrics like CPU utilization, enabling automatic scale-in or scale-out of EC2 instances.

### 8. What is CloudWatch Agent?

CloudWatch Agent collects system-level metrics (memory, disk, processes) and log files from EC2 or on-prem servers and sends them to CloudWatch.

### 9. What is CloudWatch Events / EventBridge?

It captures events generated by AWS services and routes them to targets like Lambda, SNS, or Step Functions for automation.

### 10. When would you choose CloudWatch over third-party tools?

When monitoring is AWS-centric, cost-sensitive, and requires tight integration with AWS services without managing extra infrastructure.

## ðŸ”¹ Prometheus (11â€“20)

### 11. What is Prometheus?

Prometheus is an open-source monitoring and alerting tool designed for cloud-native environments. It collects metrics using a pull-based model and stores them as time-series data.

### 12. What is the Prometheus architecture?

It consists of a Prometheus server, exporters, Alertmanager, and a time-series database. Prometheus scrapes metrics from targets at regular intervals.

### 13. What are exporters in Prometheus?

Exporters expose metrics in a Prometheus-compatible format. Examples include Node Exporter for system metrics and Kubernetes exporters.

### 14. What is PromQL?

PromQL is Prometheus Query Language used to query and aggregate time-series metrics for monitoring and alerting.

### 15. Pull vs Push model in Prometheus?

Prometheus uses a pull model where it scrapes metrics. Pushgateway is used only for short-lived jobs that cannot be scraped.

### 16. How does Prometheus handle service discovery?

Prometheus integrates with Kubernetes, Consul, EC2, and static configs to automatically discover and monitor dynamic services.

### 17. What is Alertmanager?

Alertmanager manages alerts sent by Prometheus, handling deduplication, grouping, silencing, and routing notifications to Slack, Email, PagerDuty, etc.

### 18. How is Prometheus used with Kubernetes?

Prometheus scrapes metrics from Kubernetes components like kube-api-server, nodes, pods, and services using service discovery.

### 19. What are labels in Prometheus?

Labels are key-value pairs attached to metrics, allowing filtering and aggregation (e.g., pod name, namespace, instance).

### 20. Limitations of Prometheus?

It lacks long-term storage by default, has limited native UI, and is not ideal for highly distributed multi-region monitoring without extensions.

## ðŸ”¹ Grafana (21â€“30)

### 21. What is Grafana?

Grafana is an open-source visualization tool used to create dashboards from data sources like Prometheus, CloudWatch, Elasticsearch, and Datadog.

### 22. How does Grafana work with Prometheus?

Grafana queries Prometheus using PromQL and displays the metrics in visual formats such as graphs, heatmaps, and tables.

### 23. What are dashboards in Grafana?

Dashboards are collections of panels that visualize metrics. They help teams monitor system health and performance in real time.

### 24. What are Grafana panels?

Panels are individual visual elements (graph, gauge, table) within a dashboard that represent specific metrics.

### 25. What is a Grafana data source?

A data source is where Grafana fetches data from, such as Prometheus, AWS CloudWatch, or Datadog.

### 26. How does Grafana handle alerts?

Grafana can define alert rules on panels and send notifications to Slack, Email, PagerDuty, or webhooks.

### 27. What is templating in Grafana?

Templating uses variables to dynamically filter dashboards, making them reusable across environments like dev, test, and prod.

### 28. Role-based access control in Grafana?

Grafana supports roles such as Admin, Editor, and Viewer to control access to dashboards and data sources.

### 29. Difference between Grafana and Prometheus?

Prometheus is a monitoring and data collection tool, while Grafana is a visualization and alerting platform.

### 30. Why is Grafana popular in DevOps?

It provides powerful visualizations, supports multiple data sources, and integrates well with cloud-native and DevOps tools.

## ðŸ”¹ Datadog (31â€“40)

### 31. What is Datadog?

Datadog is a SaaS-based monitoring and observability platform that provides infrastructure monitoring, APM, logs, security, and real-time dashboards.

### 32. How does Datadog collect metrics?

Datadog uses agents installed on hosts or containers to collect metrics, logs, and traces, and sends them to Datadog servers.

### 33. Datadog vs CloudWatch?

CloudWatch is AWS-native, while Datadog provides multi-cloud visibility, advanced dashboards, APM, and log correlation in a single platform.

### 34. What is APM in Datadog?

Application Performance Monitoring tracks request latency, errors, and traces across microservices, helping identify performance bottlenecks.

### 35. What is Datadog Agent?

The Datadog Agent is a lightweight service installed on hosts to collect system metrics, logs, traces, and security data.

### 36. How does Datadog integrate with Kubernetes?

Datadog provides cluster-level visibility, pod metrics, logs, and distributed tracing with minimal configuration.

### 37. What are monitors in Datadog?

Monitors define alerting rules based on metrics, logs, or traces and notify teams when conditions are met.

### 38. What is log correlation in Datadog?

Datadog correlates logs, metrics, and traces, allowing faster root cause analysis by linking all observability data.

### 39. Advantages of Datadog?

It is easy to set up, supports multi-cloud, provides unified observability, and reduces operational complexity.

### 40. When should you use Datadog?

When you need enterprise-grade, multi-cloud monitoring with minimal setup, advanced APM, and centralized observability.

## ðŸ”¹ AWS CloudWatch (41â€“50)

### 41. What is a CloudWatch namespace?

A namespace is a container for CloudWatch metrics. AWS services use predefined namespaces (e.g., AWS/EC2), while custom metrics use user-defined namespaces.

### 42. What is metric resolution in CloudWatch?

Metric resolution defines how frequently data points are recorded. Standard resolution is 5 minutes, while high-resolution metrics support 1-second granularity.

### 43. How do you monitor EC2 memory usage in CloudWatch?

EC2 does not provide memory metrics by default. You must install the CloudWatch Agent and publish memory metrics as custom metrics.

### 44. What is CloudWatch Composite Alarm?

A composite alarm combines multiple alarms using logical conditions (AND/OR), reducing alert noise and false positives.

### 45. Difference between CloudWatch Logs and S3 logs?

CloudWatch Logs are used for real-time log monitoring and alerting, while S3 logs are stored for long-term retention and analysis.

### 46. What is CloudWatch Contributor Insights?

Contributor Insights helps identify top contributors to performance issues, such as which IP addresses generate the most traffic or errors.

### 47. How does CloudWatch help in troubleshooting?

By analyzing metrics, querying logs, reviewing alarms, and correlating events, CloudWatch helps identify performance bottlenecks and failures.

### 48. What is the retention period for CloudWatch logs?

CloudWatch Logs can be retained indefinitely or configured for custom retention periods such as 7, 30, or 365 days.

### 49. What is CloudWatch Synthetics?

CloudWatch Synthetics allows you to create canaries that simulate user behavior to monitor application endpoints and APIs.

### 50. How do you reduce CloudWatch costs?

Use metric filters selectively, optimize log retention, avoid unnecessary custom metrics, and use aggregation instead of high-resolution metrics.

## ðŸ”¹ Prometheus (51â€“60)

### 51. What is a time-series database?

A time-series database stores data points indexed by time, allowing efficient querying and analysis of metrics over time.

### 52. What is scraping interval in Prometheus?

Scraping interval defines how often Prometheus collects metrics from targets. Short intervals provide more granularity but increase load.

### 53. How do you secure Prometheus?

Use TLS, authentication via reverse proxies, network restrictions, and RBAC (in Kubernetes) to secure access to Prometheus endpoints.

### 54. What is federation in Prometheus?

Federation allows a central Prometheus server to scrape metrics from multiple Prometheus servers, useful for large-scale environments.

### 55. How does Prometheus store data?

Prometheus stores metrics locally in a time-series database using blocks on disk with efficient compression.

### 56. What happens if a target is down?

Prometheus marks the target as DOWN, stops receiving metrics, and can trigger alerts based on missing data.

### 57. What is a recording rule?

Recording rules precompute frequently used or complex queries and store the results as new metrics for better performance.

### 58. How does Prometheus handle high cardinality?

Prometheus struggles with high cardinality. Best practice is to limit label combinations and avoid dynamic labels like user IDs.

### 59. What is Thanos in Prometheus?

Thanos extends Prometheus for long-term storage, high availability, and global querying using object storage like S3.

### 60. Prometheus vs CloudWatch?

Prometheus is open-source and flexible for Kubernetes, while CloudWatch is AWS-managed and tightly integrated with AWS services.

## ðŸ”¹ Grafana (61â€“70)

### 61. What is Grafana Loki?

Loki is a log aggregation system by Grafana that indexes logs using labels instead of full-text indexing.

### 62. How does Grafana Loki differ from ELK?

Loki is simpler and cost-efficient, while ELK provides full-text search and advanced log analytics.

### 63. What are Grafana annotations?

Annotations mark important events (deployments, incidents) on graphs to correlate changes with metrics.

### 64. How do you secure Grafana dashboards?

Use authentication, RBAC, folder permissions, and restrict data source access to prevent unauthorized viewing.

### 65. What is Grafana Explore?

Explore is a debugging interface that allows ad-hoc querying of logs and metrics without creating dashboards.

### 66. What is Grafana alert rule evaluation?

Grafana periodically evaluates alert conditions based on queries and triggers notifications when thresholds are breached.

### 67. What are Grafana plugins?

Plugins extend Grafana functionality by adding data sources, panels, or apps.

### 68. Grafana vs Kibana?

Grafana focuses on visualization across many data sources, while Kibana is primarily for Elasticsearch-based log analysis.

### 69. What is a Grafana organization?

An organization is a logical grouping of users, dashboards, and data sources for multi-team isolation.

### 70. How does Grafana scale?

Grafana scales by using stateless instances behind a load balancer with a shared database for dashboards and users.

## ðŸ”¹ Datadog (71â€“80)

### 71. What is Datadog Infrastructure Monitoring?

It provides real-time visibility into servers, containers, cloud services, and system-level metrics.

### 72. What is Datadog APM tracing?

Tracing captures end-to-end request flows across services, showing latency, errors, and dependencies.

### 73. How does Datadog handle log ingestion?

Datadog Agent collects logs, processes them with pipelines, and stores them for search, alerting, and correlation.

### 74. What are Datadog dashboards?

Dashboards visualize metrics and logs using widgets, enabling real-time monitoring and analysis.

### 75. What is Datadog Watchdog?

Watchdog uses machine learning to automatically detect anomalies and surface performance issues.

### 76. What is Datadog Service Map?

Service Map visualizes service dependencies, helping understand traffic flow and bottlenecks.

### 77. How does Datadog pricing work?

Datadog pricing is based on hosts, containers, metrics, logs, and APM usage.

### 78. Datadog vs Prometheus?

Datadog is fully managed and feature-rich, while Prometheus is self-hosted and more customizable.

### 79. How do you reduce Datadog costs?

Filter unnecessary logs, reduce high-cardinality metrics, optimize retention, and use sampling.

### 80. When would you combine Prometheus and Datadog?

Prometheus can be used for Kubernetes metrics, while Datadog provides centralized dashboards, APM, and long-term observability.
