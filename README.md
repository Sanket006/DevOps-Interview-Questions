## DevOps, AWS, Linux & Monitoring Interview Questions

This repository contains curated **question & answer** style notes for preparing for **Linux, DevOps, AWS, Kubernetes, Docker, Terraform, CI/CD (Jenkins), and Monitoring/Observability** interviews.  
All content is in simple Markdown so you can easily search, edit, and revise.

---

## How to Use This Repository

- **Quick revision**: Open any `.md` file and skim questions before interviews.
- **Topic‑wise practice**: Pick the technology you want to revise (e.g., Docker, Kubernetes) and go through its dedicated file.
- **Scenario‑based prep**: Use the scenario/troubleshooting files to practice real‑world style interview questions.
- **Search specific topics**: Use your editor’s search (or `rg/grep`) across the repo to find specific concepts (e.g., "autoscaling", "prometheus").

## Suggested Study Strategy

- **Phase 1 – Fundamentals**
  - Start with `Q&A-linux.md`, `Q&A-devops.md`, and `Q&A-aws.md`.
  - Then cover `Q&A-git-github.md` and `Q&A-jenkins.md` for CI/CD.

- **Phase 2 – Tools & Platforms**
  - Revise `Q&A-docker.md`, `Q&A-kubernetes.md`, and `Q&A-terraform.md`.
  - Add `Q&A-ansible.md` if you are applying for config‑management heavy roles.

- **Phase 3 – Monitoring & Observability**
  - Go through `Q&A-cloudwatch-prometheus-grafana-datadog.md` and `Q&A-monitoring-obervability.md`.

- **Phase 4 – Scenario & Troubleshooting**
  - Practice from all `real-world-secnario-based-*` and troubleshooting files.
  - Try to **answer first without looking**, then compare with the written answers.
